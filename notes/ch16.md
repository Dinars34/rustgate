# Chapter 16: Fearless Concurrency

1. **Threads (`thread::spawn`)**: Rust runs threads simultaneously. If the main thread completes, all spawned threads are instantly killed unless we use `.join().unwrap()` to block the main thread and wait.
2. **The `move` keyword**: Forces a closure to take full ownership of the variables it captures, preventing threads from referencing dropped memory.
3. **Channels (`mpsc`)**: Allows multiple producer threads to send data down a one-way tube to a single consumer thread. The receiver acts as a blocking iterator.
4. **Shared State (`Mutex<T>`)**: A lock that ensures only one thread can access or modify the inner data at a single time, preventing data races.
5. **Thread-Safe Pointers (`Arc<T>`)**: Atomic Reference Counted pointer. It safely wraps a `Mutex` so multiple threads can own and reference the exact same lock.
